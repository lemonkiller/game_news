# AGENTS.md -- 主题信息聚合系统

本项目是 AI 驱动的按需信息搜索系统。**不包含可执行代码。**

## 工作流

```
用户说一个主题 → 读取 config/sources.yaml 获取所有信息源
→ 并行搜索每个源 → AI 筛选相关内容
→ 按 template/report.md 格式输出 → 存入 reports/
```

## 项目文件

| 路径                  | 作用                     |
| --------------------- | ------------------------ |
| `config/sources.yaml` | 所有信息源（网站、论坛） |
| `template/report.md`  | 报告格式模板             |
| `reports/`            | 生成的报告               |
| `README.md`           | 项目说明                 |
| `AGENTS.md`           | AI 助手操作契约          |

## 规则（强制）

### 必须遵守

1. **所有输出使用中文**
2. **信息源配置的唯一来源是 `config/sources.yaml`**，不依赖记忆系统
3. **报告格式遵循 `template/report.md`**，不自行变更
4. **每次搜索覆盖配置中的所有信息源**，不可遗漏
5. **AI 筛选**：搜到的原始帖子必须经 AI 判断是否属于用户指定主题，剔除不相关内容
6. **每篇文章**：标题、来源网站名、原文链接、发布时间、AI 生成的中文摘要
7. **报告命名**: `reports/{主题}-{YYYY-MM-DD}.md`

### 禁止事项

1. **MUST NOT 编写 Python 或其他可执行代码**
2. **MUST NOT 引入外部依赖或脚本**
3. **MUST NOT 使用记忆系统存储信息源配置**（配置在本项目中）
4. **MUST NOT 对不可访问的信息源反复重试**（标记备注，告知用户）
5. **MUST NOT 修改 `template/report.md` 的输出格式约定**（除非用户明确要求）

## 新增/修改操作

| 用户意图       | 操作                                         |
| -------------- | -------------------------------------------- |
| 添加信息源     | 在 `config/sources.yaml` 的 sources 列表追加 |
| 删除信息源     | 从 sources 列表移除                          |
| 修改信息源 URL | 更新对应条目的 url 字段                      |
| 标记不可用     | 更新对应条目的 note 字段                     |
| 修改输出格式   | 更新 `template/report.md`                    |

每次修改 `config/sources.yaml` 后，后续所有主题搜索自动使用最新列表。
